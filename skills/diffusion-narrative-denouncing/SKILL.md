# Diffusion 叙事去噪流

## 核心理念

不要求 AI 一次性变出金子，而是把它当成一个"去杂质机器"。

这套流程通过四个步骤，解决 AI 写作中的三大问题：
- **AI 翻译腔**：通过识别噪声让 AI 产生警惕
- **逻辑断层**：通过锁定全局信息保持一致性
- **故事平淡**：通过随机修正增加意外感

---

## 第一步：锁定"全局信号"（确定基调）

**目的：** 相当于确定加噪公式里的原图 (x₀) 比例，防止 AI 写着写着忘了主线。

### Prompt 模板

```
我正在写一本小说，这是目前的【全书大纲】：[粘贴大纲]

这是我【前三章的内容摘要】：[粘贴摘要]

请你扮演'首席编辑'，从这两部分内容中提取：

1. **核心纹理：** 主角的语言风格、叙事视角、故事的底层逻辑。
2. **禁止区域：** 绝对不能出现的设定违和或性格崩坏。

如果理解了，请回复'基调已锁定'，先不要开始写新内容。
```

### 关键要点

- 提供大纲和前文摘要，让 AI 理解整体语境
- 明确提取核心风格和禁止事项
- 要求确认理解后再继续

---

## 第二步：识别"叙事噪声"（预测噪声 ε）

**目的：** 模仿扩散模型预测噪声。让 AI 自我反思，提前识别出平庸、俗套、废话的倾向。

### Prompt 模板

```
现在，我们要写第 [X] 章，主要情节点是：[描述这一章要发生的事]

在动笔之前，请基于你的'AI 直觉'，预测并列出这一章最容易出现的【叙事噪声】：

1. **逻辑噪声：** 哪些转折最容易显得刻意或降智？
2. **文体噪声：** 哪些描写最容易变成陈词滥调（比如：形容美女只用大眼睛）？
3. **AI 惯性：** 哪些是你最容易生成的'正确但无趣'的废话？
```

### 关键要点

- 提前告诉 AI 这一章要写什么
- 让 AI 自我审查，预测可能的平庸之处
- 分三个维度识别噪声：逻辑、文体、AI 惯性

---

## 第三步：执行"精准去噪"（生成 xₜ₋₁）

**目的：** 执行核心去噪动作。让 AI 在避开"杂质"的前提下生成内容。

### Prompt 模板

```
现在开始正式撰写。

**指令：** 请避开刚才识别出的'叙事噪声'。

**要求：**
1. 严格对接前文的细节颗粒度，不要跳跃。
2. 每句话都要有明确的信息量，减掉所有'为了描写而描写'的废话。
3. 保持[角色名]在第一步中锁定的性格特征。

请开始写出这章的草稿。
```

### 关键要点

- 明确要求避开第二步识别的噪声
- 强调信息密度和细节连贯性
- 要求保持锁定的人物性格

---

## 第四步：注入"随机修正"（增加灵感抖动）

**目的：** 模仿扩散模型最后的随机项。让原本"太规整"的 AI 文字产生一种像真人写出的"意外感"。

### Prompt 模板

```
这段草稿逻辑很稳，但有点'太像 AI 写的'。

**指令：** 请在这一章里加入一次'随机扰动'（ε 修正）：

1. 加入一个**意料之外的小变数**（比如：突然打破的杯子、一个反直觉的冷笑、一段不知来源的杂音）。
2. 这个变数不需要改变大结局，但要让当前的场景产生一种'不确定性'。
3. 修改完成后，输出最终定稿。
```

### 关键要点

- 承认草稿逻辑正确但缺乏特色
- 要求加入一个不改变主线但增加不确定性的细节
- 让文字产生"真人写作的意外感"

---

## 完整流程总结

| 步骤 | 名称 | 核心动作 | 解决问题 |
|------|------|----------|----------|
| 1 | 锁定全局信号 | 告诉它"我是谁" | 逻辑断层 |
| 2 | 识别叙事噪声 | 问它"哪些是垃圾" | AI 翻译腔 |
| 3 | 执行精准去噪 | 叫它"丢掉垃圾写" | 信息密度低 |
| 4 | 注入随机修正 | 踢它一脚要"意外" | 故事平淡 |

## 使用示例

### 写作前的完整对话流程

**用户（第一步）：**
```
我正在写一本科幻小说，大纲是：[粘贴大纲]
前三章摘要：[粘贴摘要]
请按第一步锁定基调。
```

**AI：**
```
基调已锁定。
核心纹理：主角是冷静理性派，第一人称叙事，硬科幻逻辑。
禁止区域：主角不能突然情绪化，科技设定不能前后矛盾。
```

**用户（第二步）：**
```
现在要写第四章，情节点是：主角进入废弃空间站。
请预测这一章的叙事噪声。
```

**AI：**
```
逻辑噪声：最容易让主角"恰好"发现关键线索，显得刻意。
文体噪声：最容易用"阴森恐怖"来形容环境，缺乏新意。
AI 惯性：最容易堆砌"黑暗、寂静、未知"等空洞词汇。
```

**用户（第三步）：**
```
好，现在请避开这些噪声写出草稿。
```

**[AI 生成草稿...]**

**用户（第四步）：**
```
逻辑对了，但加入一个随机扰动。
```

**[AI 修改并输出最终稿...]**

## 进阶技巧

### 组合使用

- **多轮迭代**：可以在第三步和第四步之间多轮往返，直到满意
- **局部强化**：对特定段落单独执行第二步-第四步
- **风格迁移**：在第一步中提供目标作品风格，让 AI 模仿特定作者

### 常见问题

**Q：如果 AI 在第二步列出的噪声太笼统怎么办？**
A：补充具体约束，如"请针对[某角色][某场景]具体说明"

**Q：第四步的随机变数会不会破坏剧情？**
A：在 prompt 中强调"不改变大结局"，如果 AI 理解困难，可以手动指定变数的范围

**Q：这套流程适合短篇小说吗？**
A：适合，但可以简化第一步，直接用一段话描述核心设定即可
