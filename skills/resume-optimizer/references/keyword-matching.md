# 关键词匹配策略

## 关键词类型

### 1. 硬技能（Hard Skills）
- 编程语言：Python, Java, JavaScript, SQL
- 工具/平台：Tableau, Power BI, AWS, Docker
- 方法论：Agile, Scrum, Kanban, TDD

### 2. 行业术语
- 职位相关：data pipeline, ETL, dashboard, stakeholder management
- 领域相关：customer retention, churn analysis, A/B testing

### 3. 软技能
- Communication, Leadership, Problem-solving, Teamwork
- 注意：用具体例子体现，不要只列词汇

## 提取JD关键词的技巧

### 优先级排序
1. **必须匹配** - 出现在职位要求中的硬技能
2. **应该匹配** - "preferred" 或 "plus" 的技能
3. **加分项** - 可转移的相关技能

### 变体和同义词
- 同时使用缩写和全称
  - SQL / Structured Query Language
  - API / Application Programming Interface
  - KPI / Key Performance Indicator
- 使用行业常用变体
  - Data Analysis / Analytics
  - Manage / Led / Directed

## 自然融入策略

### ✅ 好的例子
```
• Developed Python scripts using Pandas and NumPy to automate data
  processing, reducing manual work by 40%
• Created interactive Tableau dashboards to visualize KPI trends for
  executive stakeholders
```

### ❌ 坏的例子（关键词堆砌）
```
• Skills: Python, Pandas, NumPy, Data Processing, Tableau, Dashboards,
  KPI, Visualization, Stakeholders, Automation, Scripts
```

## 匹配度评估

| 匹配度 | 关键词覆盖 | 建议 |
|--------|-----------|------|
| 90%+ | 几乎所有必须/应该匹配 | 直接投递 |
| 70-90% | 大部分必须匹配，少数应该匹配 | 可以投递，在Cover Letter解释差距 |
| 50-70% | 核心必须匹配，应该匹配不足 | 考虑先提升技能再投递 |
| <50% | 关键必须匹配缺失 | 不建议投递，目标不匹配 |

## 实用技巧

1. **创建技能矩阵表**
   - 列出JD所有关键词
   - 标记自己掌握程度（精通/熟练/了解/无）
   - 针对性撰写相关经历

2. **使用技能的同义词**
   - JD说"manage"，你可以说"led", "directed", "supervised"
   - JD说"develop"，你可以说"created", "built", "implemented"

3. **量化关键词相关成果**
   - 不要只说"used Python"
   - 要说"used Python to automate X, achieving Y result"
