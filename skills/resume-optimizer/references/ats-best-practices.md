# ATS优化最佳实践

## 格式标准化

### ✅ 推荐做法
- 单列布局，从上到下线性排列
- 标准章节标题：Work Experience, Education, Skills, Certifications
- 一致的日期格式：MM/YYYY 或 Month YYYY – Month YYYY
- 标准字体：Arial, Calibri, Times New Roman
- 正文字号：10-12pt
- 保存为 .DOCX 格式（除非职位明确要求PDF）

### ❌ 避免做法
- 表格、多列布局
- 文本框、图形、图标
- 页眉/页脚放置关键信息
- 图片/LOGO
- 自定义字体或颜色
- 扫描版PDF（必须是可搜索文本）

## 章节顺序

1. **联系方式** - 姓名、电话、邮箱、LinkedIn（可选）
2. **专业总结** - 3-4行概述核心能力
3. **技能** - 技术技能、工具、软技能
4. **工作经历** - 倒序排列，最近的最详细
5. **教育背景** - 学位、专业、学校、时间
6. **证书** - 仅列出相关的

## 内容原则

### 工作经历Bullet Points

每个bullet使用 **STAR+T** 格式：
- **S**ituation - 情境背景
- **T**ask - 任务目标
- **A**ction - 采取的行动（使用强力动词）
- **R**esult - 结果（量化！）
- **T**ool - 使用的工具/技术

示例：
```
• Developed SQL queries to analyze 1M+ user behavior records,
  identifying churn patterns and driving 15% improvement in customer retention
  through targeted intervention campaigns (Python, Tableau)
```

### 强力动词列表

**分析类**: Analyzed, Evaluated, Assessed, Examined, Investigated
**创造类**: Developed, Designed, Created, Built, Implemented
**改进类**: Optimized, Enhanced, Improved, Streamlined, Automated
**领导类**: Led, Managed, Supervised, Directed, Coordinated
**成就类**: Achieved, Delivered, Completed, Executed, Launched

## ATS解析测试

在投递前，使用以下工具测试：
- Jobscan (jobscan.co)
- Resume Worded (resumeworded.com)
- Resume.io ATS Checker

测试重点：
1. 解析是否正确（信息未丢失/乱序）
2. 关键词匹配度
3. 格式兼容性
